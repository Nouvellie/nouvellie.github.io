Creation of an iOS app with Swift and Alamofire 5, for the integration of Django APIs The app has a SignIn via Alamofire request Django Login Api, where it gets a token that is stored in UserDefault with SuiteName, which is then sent as a Header for requests to the main APIs. These core API calls are made via a ShareExtension, which receives files from Gmail, Webs, or iCloud, etc. Fully integrated and associated to the same UserDefault, with corresponding validations. These calls are only processed by the remote Django API if the token is valid. It then renders the response received casted to JsonResponse, on screen. While all this happens for background, in the app we'll see a custom loading that ends in a custom progress bar adapted to a percentage of result (0-100%). Transition of views, through buttons or Api response validations. On the other hand, there is a menu that has options switch, both of inputs details for the APIs, and also extra data generation, that are sent as a dictionary to the remote API. Internally, a function was made with Localizable Strings, which adapts all the language of both the app and the API response to the language of the phone itself.

Geometric Meshes: Algorithm

Improvement of the quality of the 2D geometric meshes of the Delaunay triangles, by means of criteria in the limit of the angles, refining Africa of 9665 triangles. It started as a project in c and c++, which became quite complicated, to then continue with Java, and finally we decided to do everything from scratch keeping the logic using and learning Python at the same time, given the ease provided by this programming language with its libraries. The project was at the level of our teacher's thesis in the course of Parallel Computing (tenth semester).
